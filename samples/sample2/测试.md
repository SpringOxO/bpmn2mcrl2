# sample2 手动测试说明

本文档用于复现 `samples/sample2/camunda` 下四个 BPMN 文件的转换测试，并解释每一步为什么算成功。

## 测试目标

输入目录：

```text
samples/sample2/camunda
```

输出目录：

```text
samples/sample2/manual-test-output
```

覆盖的 BPMN 文件：

```text
feature.bpmn
loan-granting.bpmn
order-handling.bpmn
taskAssignmentEmail.bpmn
```

需要验证的能力：

| 能力 | 覆盖文件 | 成功标志 |
| --- | --- | --- |
| `userTask` 支持 | `feature.bpmn`, `taskAssignmentEmail.bpmn` | 输出业务动作名，不再输出 `usertask_1` |
| `exclusiveGateway` 支持 | `feature.bpmn`, `loan-granting.bpmn` | 输出 `tau . A + tau . B` 选择结构 |
| `subProcess` 展开 | `loan-granting.bpmn`, `order-handling.bpmn` | 输出子流程内部动作，不再输出子流程 ID 占位 |
| `boundaryEvent` 支持 | `order-handling.bpmn` | 输出 `boundary_archive_invoice` 和 `archive_invoice` |
| mCRL2 语法有效 | 四个输出文件 | `mcrl22lps` 返回 `exit=0` |

## 0. 进入项目目录

在 PowerShell 中执行：

```powershell
cd D:\sf\2026spring\并发理论\Project\bpmn2mcrl2
$env:PYTHONIOENCODING='utf-8'
```

说明：

- 后续命令都假定当前目录是 `bpmn2mcrl2`。
- `PYTHONIOENCODING='utf-8'` 用于保证 Python 控制台输出中文和符号时不乱码。

## 1. Python 语法检查

命令：

```powershell
python -c "from pathlib import Path; compile(Path('scripts/bpmn2mcrl2.py').read_text(encoding='utf-8'), 'scripts/bpmn2mcrl2.py', 'exec'); print('syntax ok')"
```

预期输出：

```text
syntax ok
```

成功原因：

- `compile()` 会让 Python 读取并编译 `scripts/bpmn2mcrl2.py`。
- 如果脚本存在缩进错误、括号缺失、字符串未闭合等语法问题，这一步会直接报错。
- 输出 `syntax ok` 表示脚本本身是合法 Python 文件。

注意：这里不使用 `python -m py_compile`，因为它会写入 `__pycache__`，在某些环境下可能因为缓存目录权限导致误报失败。

## 3. 关键语义检查

命令：

```powershell
$checks = @(
  @('feature.mcrl2', 'handle_feature_request', 'present'),
  @('feature.mcrl2', 'usertask_1', 'absent'),
  @('taskAssignmentEmail.mcrl2', 'do_something', 'present'),
  @('taskAssignmentEmail.mcrl2', 'usertask_1', 'absent'),
  @('loan-granting.mcrl2', 'grant_loan', 'present'),
  @('loan-granting.mcrl2', 'reject_loan_request', 'present'),
  @('loan-granting.mcrl2', 'subprocess_16kr5xn', 'absent'),
  @('order-handling.mcrl2', 'create_invoice', 'present'),
  @('order-handling.mcrl2', 'archive_invoice', 'present'),
  @('order-handling.mcrl2', 'boundary_archive_invoice', 'present'),
  @('order-handling.mcrl2', 'task_1nbdup3', 'absent')
)

$failed = $false
foreach ($check in $checks) {
  $file = $check[0]
  $token = $check[1]
  $mode = $check[2]
  $text = Get-Content -Raw -Encoding UTF8 (Join-Path 'samples\sample2\manual-test-output' $file)
  $hasToken = $text.Contains($token)
  $ok = if ($mode -eq 'present') { $hasToken } else { -not $hasToken }
  Write-Host ("{0,-28} {1,-28} {2,-7} -> {3}" -f $file, $token, $mode, $(if ($ok) { 'OK' } else { 'FAIL' }))
  if (-not $ok) { $failed = $true }
}

if ($failed) {
  throw 'Semantic checks failed.'
}
```

预期输出：

```text
feature.mcrl2                handle_feature_request       present -> OK
feature.mcrl2                usertask_1                   absent  -> OK
taskAssignmentEmail.mcrl2    do_something                 present -> OK
taskAssignmentEmail.mcrl2    usertask_1                   absent  -> OK
loan-granting.mcrl2          grant_loan                   present -> OK
loan-granting.mcrl2          reject_loan_request          present -> OK
loan-granting.mcrl2          subprocess_16kr5xn           absent  -> OK
order-handling.mcrl2         create_invoice               present -> OK
order-handling.mcrl2         archive_invoice              present -> OK
order-handling.mcrl2         boundary_archive_invoice     present -> OK
order-handling.mcrl2         task_1nbdup3                 absent  -> OK
```

成功原因：

- `handle_feature_request` 和 `do_something` 出现，说明 `userTask` 已按 BPMN 的 `name` 生成动作。
- `usertask_1` 不出现，说明用户任务没有退化成节点 ID。
- `grant_loan` 和 `reject_loan_request` 出现，说明 `loan-granting.bpmn` 的子流程内部被展开。
- `subprocess_16kr5xn` 不出现，说明子流程没有被偷懒建模为一个占位动作。
- `create_invoice` 出现，说明 `order-handling.bpmn` 的子流程内部服务任务被展开。
- `boundary_archive_invoice` 和 `archive_invoice` 出现，说明边界事件及其后续处理分支进入了模型。
- `task_1nbdup3` 不出现，说明 `order-handling.bpmn` 的子流程没有被当成节点 ID 占位。

## 4. mCRL2 线性化检查

命令：

```powershell
Get-ChildItem samples\sample2\manual-test-output -Filter *.mcrl2 | Sort-Object Name | ForEach-Object {
  $lps = [IO.Path]::ChangeExtension($_.FullName, ".lps")
  mcrl22lps $_.FullName $lps
  Write-Host "$($_.Name) exit=$LASTEXITCODE"
}
```

预期输出：

```text
feature.mcrl2 exit=0
loan-granting.mcrl2 exit=0
order-handling.mcrl2 exit=0
taskAssignmentEmail.mcrl2 exit=0
```

成功原因：

- `mcrl22lps` 会读取 `.mcrl2` 文件并转换成 LPS。
- 它会检查 mCRL2 语法、动作声明、类型参数、进程初始化等内容。
- `exit=0` 表示 mCRL2 工具接受该文件。

可能看到的提示：

```text
The action Terminate followed by a deadlock is added to signal termination of the linear process.
```

这是 mCRL2 对有限终止进程的正常提示，不是失败。判断标准仍然是 `exit=0`。
